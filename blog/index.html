Augmented Lagrangian Method: New Releases!

October 14, 2019

Pantelis Sopasakis

OpEn can now solve problems of the very general form

\[\begin{split}\mathbb{P}(p) {}:{} \operatorname*{Minimize}_{u {}\in{} \mathbb{R}^{n_u}}&\ \ f(u, p)\\ \mathrm{subject\ to}\ \ &u \in U\\ & F_1(u, p) \in C\\ & F_2(u, p) = 0\end{split}\]

where

  • $f$ is a smooth cost functions
  • $U$ is a simple, possibly nonconvex set
  • $F_1$ and $F_2$ are smooth mappings
  • $C$ is a convex set

Constraints $F_1(u, p) \in C$ are treated with the augmented Lagrangian method and constraints $F_2(u, p) = 0$ with the penalty method.

This new functionality is supported in Rust (version > 0.6.0) and Python (version > 0.2.0b.2).

Python interface

June 3, 2019

Pantelis Sopasakis

The new Python interface is finally out and the solver now supports generic constraints of the form

\[\begin{split} \operatorname*{Minimize}_{u {}\in{} \mathbb{R}^{n_u}}& \ f(u; p)\\ \mathrm{subject\ to}\, &u \in U, \\ &c(u; p) = 0,\end{split}\]

where $c:\mathbb{R}^{n_u}\times \mathbb{R}^{n_p} \to \mathbb{R}^{n_c}$.

This can be used to encode either equality or inequality constraints of the form $\psi(u; p) \leq 0$ using

\[c(u; p) = \max\{0, \psi(u; p)\}.\]

The Python package is available on pip. To install it just type in your terminal:

pip install opengen

Read more about this here.

Rust for robots

March 19, 2019

Pantelis Sopasakis

Is the Rust programming language the future of robotics? Some claim that its memory and thread safety model and the fact that it can run on embedded and bare metal devices makes it the perfect candidate for robotics.

Despite its young age, Rust has all but failed to go unnoticed: a stackoverflow survey revealed that Rust is the top most loved programming language and nowhere around the 25 most dreaded ones.

Rust comes with all the perks of high-level languages (such as C++ and Java) related to code structure and organisation, the capabilities and efficiency of low-level languages (such as C), plus it is tRustworthy.

This is why we decided to write OpEn in Rust.
